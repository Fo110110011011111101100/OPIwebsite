Разработка алгоритмов работы модулей программы

Действия, которые можно осуществлять в программе

При запуске программы отображается начальный интерфейс и приветствие, после запрашивается логин и пароль. Если введенный логин и пароль верны, то выводится или администраторский, или пользовательский интерфейс соответственно и появляется возможность дальнейшего взаимодействия с системой.

Действие «добавить учетную запись»

Выбирается соответствующий пункт меню. Для того, чтобы добавляемая учетная запись сохранилась, должен открыться файл в режиме записи. Далее программа запрашивает ввод логина. Вводится пароль и логин нового пользователя, выбор статуса. После этого завершается запись в файле, выводится сообщение о успешно созданной записи, данные сохраняются и файл закрывается.

Действие «удалить учетную запись»

На консоль выведется таблица с данными об учетных записях и будет запрошен табельный номер учетной записи для удаления. Вводится табельный номер и, если удаление подтверждается, то запись удаляется из файла и сообщается о ее успешном удалении, если удаление отклоняется, выводится сообщение об отмене удаления и данные в файле остаются прежними.

Редактирование учетных записей пользователей.

Далее из меню выбирается пункт «отредактировать учетную запись». Открывается файл с учетными записями. Затем необходимо ввести логин учетной записи, которую следует изменить. После заполняются новые данные и измененные данные сохраняются в учетную запись.

Добавления записи

Для того, чтобы запись добавилась, необходимо открыть файл для записи. Затем последовательно вводятся ФИО клиента, год, месяц оказания услуги, наименование и ее стоимость. Чтобы введенные данные сохранились, необходимо закрыть файл, то есть завершить запись.

Действия «сортировка»

Выбрать пункт «сортировка» из меню и решить, по какому критерию она будет производиться. В зависимости от критерия данные будут выводиться или в алфавитном порядке (для текстовых данных), или в порядке возрастания (для числовых данных).

Последовательность для Администратора

После запуска программы происходит авторизация, заключающаяся в вводе логина и пароля. В управлении учётными записями пользователя можно: добавить, посмотреть, отредактировать и удалить данные о пользователе. При добавлении пользователя происходит запрос на ввод данных и подтверждение добавления, при редактировании запрос на логин пользователя и повторном вводе данных и подтверждения редактирования, при отсутствии, пишет, что такого пользователя нет, при удалении запрос на ввод логина и подтверждения удаления данного пользователя, при просмотре данных выводит таблицу с данными. При выборе работе с файлами данных можно создать файл, удалить, открыть. При создании запрос на ввод имени файла и подтверждения об его добавлении ,при удалении запрос на ввод имени файла и подтверждения удаления ,при открытии запрос на ввод имени файла ,при вводе имени несуществующего файла для удаления или редактирования ,выведет сообщение такого файла не существует .при выборе пункта работа с данными можно добавить , посмотреть, редактировать ,удалить данные .При просмотре выведется табличка со всеми данными .При удалении произойдет запрос на ввод табельного номера сотрудника ,и подтверждение на удаление сотрудника ,если такой имеется ,в случае редактирования также произойдёт запрос на ввод табельного номера с последующим заполнением новых данных. При выборе пункта выход из учётной записи, произойдёт возврат в главное меню.

Последовательность для Пользователя

После запуска программы происходит авторизация, при правильном вводе происходит выбор пунктов, просмотр данных (выводятся все данные водителей), подсчет суммы оплаты конкретному сотруднику (запрос на ввод табельного номера и вывод результата), поиск и сортировка по полям.

Варианты использования

Администратор имеет такие возможности как: управления учётными записями, управления данными (просмотреть все данные, добавить запись, удалить запись, редактировать), работу с файлами. Пользователь имеет такие возможности как: выполнить сортировку, просмотр всех данных, определить объём выплат конкретному сотруднику за определённый период времени, выполнить поиск по фамилии и табельному номеру.

Описание программного кода

Библиотека iostream предназначена для ввода-вывода в консоли. Все возможности данного заголовочного файла размещены в пространстве имен std, поэтому для использования этих возможностей необходимо прописать пространство имен через using namespace. Заголовочный файл fstream предоставляет функционал для считывания данных из файла и записи данных в файл. Реализуемые с помощью этой библиотеки возможности: операторы ввода << и вывода >>:void методы записи и чтения строк getline(), put():void потоковая запись и чтение методами write() и read():void методы открытия и закрытия файлов, open() и close():void методы проверки открыт ли файл is_open() и достигнут ли конец файла eof(). Данный заголовочный файл включает в себя класс ifstream, предоставляющий возможности чтения файлов. Метод close() закрывает файл. Работать с динамическими массивами с помощью выделения и очистки области памяти не так удобно, как с векторами. Если необходимо использовать векторы в программе, необходимо подключить заголовочный файл . Для добавления нового элемента в конец вектора используется метод push_back(). Количество элементов определяется методом size(). Для доступа к элементам вектора можно использовать квадратные скобки [], также, как и для обычных массивов. Метод pop_back() используется для удаления последнего элемента, empty() — проверяет вектор на пустоту. Библиотека conio.h подключена для того, чтобы использовать функцию getch(), которая читает один символ с клавиатуры, но не выводит его на экран. Cstdlib – заголовочный файл, который определяет несколько функций общего назначения, например, для динамического управления памятью, генерации случайных чисел, целой арифметики, поиска, сортировки и преобразования типов данных. В программе используется одна из функций этой библиотеки, puts(), которая выводит строку на стандартный поток вывода (на экран). Для реализации возможности работы со строковым типом данных string необходимо подключить заголовочный файл string. Sstream — заголовочный файл с классами, функциями и переменными для организации работы со строками. Библиотека Windows.h подключается для того, чтобы обеспечивать корректный ввод-вывод кириллицы с помощью функции SetConsoleCP(), которая устанавливает нужную кодовую таблицу, на поток ввода, и функции SetConsoleOutputCP(), которая устанавливает нужную кодовую таблицу, на поток вывода. Для использования манипуляторов, например, setw (int w), который устанавливает ширину поля w, в программу необходимо включить заголовочный файл iomanip.h.

Следующей идет пользовательская функция приветствия, возвращающая текущую дату и время, содержащая информацию о разработчике, руководителе проекта и годе разработки. Вывод текущей даты и времени осуществляется с помощью структуры time_t, члены которой используются для месяца, дня, года, дня недели, часа, минут, секунд и миллисекунд и функции localtime(&now). Пользовательская функция currentDateTime() осуществляет возможность вставки на страницу текущей даты и времени. Функция hi () позволяет выводить информацию о разработчике, руководителе и годе разработки. Далее описаны классы User и Admin (см. рисунок 2.2.1). Подробное описание этих классов было приведено ранее.

Далее создаться объекты классов User и Admin, а после объявляется вектор vector<User> users для хранения списка пользователей.

Функция startAdmin() – запускает меню администратора, в которой с помощью конструкции switch case выбираются пункты меню и вызываются соответствующие функции в случае если текущая запись администраторская. Функция showusermenu() – меню пользователя, в которой также выбираются пункты меню и вызываются функции в случае если текущая запись пользовательская.
